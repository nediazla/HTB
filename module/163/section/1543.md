Mientras tanto, hemos estado ejecutando un escaneo de puerto completo utilizando el`-A`bandera ([Opciones de escaneo agresivas](https://nmap.org/book/man-misc-options.html)) para realizar una enumeración adicional, incluida la detección del sistema operativo, el escaneo de versiones y el escaneo de script. Tenga en cuenta que este es un escaneo más intrusivo que solo correr con el`-sV`Indicador para el escaneo de versiones, y debemos tener cuidado de asegurarnos de que cualquier scripts que se ejecute con el escaneo de script no cause ningún problema.

Lo primero que podemos ver es que este es un host Ubuntu que ejecuta un proxy HTTP de algún tipo. Podemos usar este práctico nmap grep[hoja de trucos](https://github.com/leonjza/awesome-nmap-grep)para "cortar el ruido" y extraer la información más útil del escaneo. Para Fuzz Vhosts, primero debemos averiguar cómo se ve la respuesta para un VHOST inexistente. Podemos elegir todo lo que queramos aquí; Solo queremos provocar una respuesta, por lo que debemos elegir algo que muy probablemente no exista.

  Recopilación de información externa

```shell-session
xnoxos@htb[/htb]$ curl -s -I http://10.129.203.101 -H "HOST: defnotvalid.inlanefreight.local" | grep "Content-Length:"

Content-Length: 15157
```

Intentando especificar`defnotvalid`en el encabezado del host nos da un tamaño de respuesta de`15157`. Podemos inferir que esto será lo mismo para cualquier vhost inválido, así que trabajemos con`ffuf`, usando el`-fs`indicando para filtrar las respuestas con tamaño`15157`ya que sabemos que son inválidos.
